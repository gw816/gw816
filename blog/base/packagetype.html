第7节：包装类型

对基本数据类型有了解以后可以做一个测试：

package cc.igavin.demo;

/**
 * 有public修改的类
 * @author Gavin[igavin.cc]
 * @since 2019-06-17 18:54
 */
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World");
        People people=new People();
        people.setName("Gavin");
        people.setAge(30);
        people.sayHelp();
    }
}

/**
 * 一个java文件中只能一个pubLic修饰的类，可以外部访问，没有修改的只能在本包中访问。
 * 通常不建议这样一个文件中多个类，这样写只是为了方便
 * @author Gavin[igavin.cc]
 * @since 2019-06-17 18:59
 */
class People{
    private String name;
    private int age;
    public void setName(String name){
        this.name=name;
    }
    public void setAge(int age){
        this.age=age;
    }
    public void sayHelp(){
        System.out.println(name+","+age+":Help!");
    }
}

输出： Hello World Gavin,30:Help!

给People类加一个年龄的属性，类型是int，测试会发现这个age会永远有值(默认值0)，那可不可以存在这种情况，年龄是int，但是不知道具体的值，在确定知道以后再设值？

针对这样的问题，java提供了基本数据类型的包装类型。

基本类型 二进制位数 包装器类 boolean 1 Boolean byte 8 Byte char 16 Character short 16 Short int 32 Integer long 64 Long float 32 Float double 64 Double

修改年龄为Integer类型：

package com.sxf.ucenter.sync.handler;

/**
 * 有public修改的类
 * @author Gavin[igavin.cc]
 * @since 2019-06-17 18:54
 */
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World");
        People people=new People();
        people.setName("Gavin");
//        people.setAge(30);
        people.sayHelp();
    }
}

/**
 * 一个java文件中只能一个pubLic修饰的类，可以外部访问，没有修改的只能在本包中访问。
 * 通常不建议这样一个文件中多个类，这样写只是为了方便
 * @author Gavin[igavin.cc]
 * @since 2019-06-17 18:59
 */
class People{
    private String name;
    private Integer age;
    public void setName(String name){
        this.name=name;
    }
    public void setAge(int age){
        this.age=age;
    }
    public void sayHelp(){
        System.out.println(name+","+age+":Help!");
    }
}

输出： Hello World Gavin,null:Help!

自动装箱拆箱

基本数据类型在和对应的包装类型操作时，包括赋值、比较等，在编译阶段，编译器会自动在代码拼上Integer.valueOf(int)或者Integer.intValue()方法。

Integer i = 10;  //装箱，=后面编译在Integer.valueOf(10)
int n = i;   //拆箱，=后面编译成i.intValue()

引申

Integer a1=new Integer(30);
Integer a2=new Integer(30);
Integer b1=30;
Integer b2=30;
System.out.println(a1==a2);
System.out.println(a1==b1);
System.out.println(b1==b2);
System.out.println(a1.equals(b1));

输出结果： false false true true

想知道原因请自行查看Integer源码的Integer.valueOf(int)方法，里面用到了缓存，b1和b2实际是指向同一个对象。
