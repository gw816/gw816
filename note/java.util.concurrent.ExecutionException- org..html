看完这段异常，百思不得其解，网上一查，说是web.xml，的一个地方写错的可能性很大，说什么<url-pattern>中要怎么样，直接被我忽视，因为这项目碰都没有碰过web.xml，于是，第一直觉就想到了最近添加的jaxen包，但是也没什么冲突，我就把服务器上的项目中lib目录和本地lib对比了一下，我擦，直接比原来的lib多出了8M，这些包都是maven根据依赖关系自动导入的，我就挨个排除jar,最后认定是这些jar包中的之一影响了web.xml的解析：

icu4j-2.6.1.jar 、 jdom-1.0.jar、xalan-2.6.0.jar、xercesImpl-2.8.1.jar、

xml-apis-1.3.02.jar、xmlParserAPIs-2.6.2.jar、xom-1.0.jar

删掉这些包后，项目正常运行，不得不感叹maven，帮了倒忙，还得排查问题，深沉原因也没具体分析，只知道目前项目的xml解析方式和新导入的xml解析有冲突

另外一个解决方式是把web.xml中的

<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xmlns="http://java.sun.com/xml/ns/javaee" xmlns:web="http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"
xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_3_0.xsd"
id="WebApp_ID" version="3.0">

改成

<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xmlns="http://java.sun.com/xml/ns/javaee" xmlns:web="http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"
xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"
id="WebApp_ID" version="2.5">
也可以解决tomcat启动报上面异常的问题，但是目前没有测试别的功能是否会被影响，妥当的就是先删掉那些包把，第二种在测试下

记在这里，希望其他同学也注意下，网上这样的异常千篇一律的解释说web.xml参数设置错误，还请大家具体问题具体分析，共同进步！！
